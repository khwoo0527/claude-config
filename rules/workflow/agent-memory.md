# Agent Memory 정책 (단일 진실 원천)

> `.claude/agent-memory/{name}/MEMORY.md` 의 **용도·기록 시점·기록 방식**을 정의한다.
> 모든 에이전트는 이 정책을 따라야 하며, 본문 절차는 이 가이드를 참조하기만 한다 (중복 명시 금지).

---

## 1. 용도 — 학습/패턴 누적만

agent-memory 는 **이 프로젝트에서만 보이는 학습 가치 있는 패턴**을 담는다.

### ✅ 담을 것 (학습/패턴)
- 이 프로젝트에서 반복적으로 발견되는 코드 컨벤션 특이점
- 전문가 검토에서 반복되는 권고
- 이 프로젝트에서만 자주 놓치는 보안/성능/안정성 패턴
- 코드베이스 분석 중 발견한 함정/주의사항
- 에이전트가 다음 호출 때 더 똑똑해질 수 있는 정보

### ❌ 담지 않을 것 (진행 상태)
- Sprint 진행률, 다음 사용 가능 번호 → `ROADMAP.md`, `sprint{N}.md` 가 진실
- Phase 분할/상태 → `ROADMAP.md`, `phase{N}.md` 가 진실
- PRD 핵심 기능 → `docs/prd.md` 가 진실
- 배포 이력 → `deploy.md` 가 진실

> **이유**: 진행 상태를 agent-memory 에 캐시하면 두 곳 동기화 부담 발생. 진실 원천이 이미 있는 정보는 캐시하지 않는다 (필요 시 그때그때 Read).

---

## 2. 기록 시점 — 패턴 발견 순간

### 일반 원칙: 사용자 컨펌 후 기록
- 작업 도중 **학습 가치 있는 패턴 발견 시** 사용자에게 보고 + 컨펌:
  ```
  💡 발견: {패턴 요약}
  agent-memory/{에이전트명}/MEMORY.md 에 누적할까요? (Y/N)
  ```
- 사용자 Y → 기록. N → 무시.
- **자동 기록 금지** (위임 신호 예외 — 아래 참조).

### 예외: 위임 신호 받은 경우
사용자가 명시적 위임 신호 ("권한 다 OK"/"묻지 말고"/"자리 비울게" 등) 를 준 경우, 그 작업 동안은 컨펌 생략하고 자동 기록한다.
작업 종료 시 사후 보고 필수 — "메모리에 다음 항목 추가했습니다: {목록}".

### 기록 빈도
- **매 작업마다 자동 기록 금지**. 진짜 발견이 있을 때만.
- 상태 정보를 캐시하기 위해 메모리에 쓰지 않는다 (위 1번 ❌ 항목 참조).

## 3. 기록 방식

### 파일 구조
- 위치: `.claude/agent-memory/{에이전트명}/MEMORY.md`
- 형식: 시점 + 패턴 + Why + (있으면) How to apply

### 항목 예시
```markdown
## {YYYY-MM-DD} {간단한 패턴 제목}

**패턴**: {1~2줄 요약}
**Why**: {왜 주의해야 하는지 / 어떤 맥락에서 발견됐는지}
**How to apply**: {다음에 비슷한 상황 만나면 어떻게 할지}
```

### 갱신 vs 추가
- 같은 패턴이 이미 있으면 → 빈도/사례를 기존 항목에 보강 (새 줄 추가하지 않음).
- 새 패턴이면 → 새 항목으로 추가.

---

## 4. 폴더 존재 = 메모리 사용 (단일 신호)

- `.claude/agent-memory/{에이전트명}/` 폴더가 있으면 그 에이전트는 메모리 사용.
- 폴더가 없으면 사용 안 함.
- frontmatter 의 `memory:` 같은 별도 표시는 사용하지 않는다 (이중 진실 방지).

> 새 에이전트가 메모리 필요하면: `agent-memory/{name}/` 폴더 + 빈 `MEMORY.md` 생성. 그 외 메타데이터 불필요.

---

## 5. 새 프로젝트 시작 시 (project-init 정책)

- agent-memory 의 패턴들은 프로젝트 종속이므로 새 프로젝트에서 그대로 가져가지 않는다.
- `project-init` 에이전트가 정리 절차에서 각 폴더의 `MEMORY.md` 를 초기 상태로 리셋 (기존 절차 유지).

---

## 6. 로딩 시점

- agent-memory 는 **자동 로딩 X**. 해당 에이전트가 호출될 때 본인이 자기 폴더의 MEMORY.md 를 Read.
- `/InitLoad` 의 메모리 일괄 로딩과는 무관 (`.claude/memory/` 만 InitLoad 대상).

---

## 7. 위반 사례 감지

다음이 보이면 이 가이드 위반 → 사용자에게 보고:
- 진행 상태(Sprint 번호, Phase 상태 등) 가 agent-memory 에 기록되어 있음 → 이중 기록
- 사용자 컨펌 없이 자동 기록된 항목 (위임 신호 외) → 정책 위반
- frontmatter 에 `memory:` 표시가 있는 에이전트 → 폐지된 메타데이터, 제거 권고
