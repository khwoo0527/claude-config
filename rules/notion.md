# Notion 기술 문서 관리

> 이 파일은 Notion을 기술 문서 허브로 사용하는 프로젝트에 적용되는 규칙입니다.
> 프로젝트별 Notion 페이지 구조와 연동 설정은 `/CLAUDE.md`에 정의합니다.

## 설정 방법

프로젝트에서 Notion 연동을 사용하려면 `/CLAUDE.md`에 다음을 정의합니다:

```markdown
## Notion 연동
- 루트 페이지: {Notion 페이지 URL}
- Integration 토큰: 환경 변수 `NOTION_TOKEN`으로 관리
- API 버전: 2022-06-28

### 하위 페이지 (page ID)
- 프로젝트 개요: `{page-id}`
- 기능 명세: `{page-id}`
- 릴리즈 노트: `{page-id}`
- (프로젝트에 맞게 추가)
```

## Notion API 사용 규칙

- Integration 토큰은 **절대 소스 코드에 하드코딩하지 않음**
- 토큰 관리 방식은 프로젝트별로 결정 (환경 변수, 시크릿 매니저, 설정 파일 등)
- `.claude/` 폴더는 git에 포함하여 멀티 환경 동기화 보장 — `.gitignore`에 넣지 않음
- 토큰이 `.claude/` 내 파일에 포함된 경우, 안전한 관리 방법을 별도로 수립
- API 호출 시 rate limit 주의 (Notion API: 3 requests/second)

## 문서 업데이트 트리거

> 아래는 일반적인 트리거 목록입니다. 프로젝트별로 `/CLAUDE.md`에서 커스터마이징합니다.

| 이벤트 | 업데이트 대상 |
|--------|-------------|
| 새 버전 배포 | 릴리즈 노트 |
| API 변경/추가 | 기능 명세 |
| 새 기능 추가 | 기능 명세 |
| 아키텍처 변경 | 시스템 아키텍처 |
| 스프린트 시작/완료 | 개발 프로세스 |
| 데이터 모델 변경 | 데이터 모델 |

## Notion 페이지 구조 권장안

```
{프로젝트명} (루트)
├── 프로젝트 개요
├── 시스템 아키텍처
├── 기능 명세
├── 데이터 모델
├── 개발 프로세스
│   ├── 로드맵
│   └── 스프린트 기록
├── 릴리즈 노트
└── 운영 가이드 (해당 시)
```

## 에이전트 연동

- 에이전트가 Notion 업데이트를 안내할 때 `/CLAUDE.md`에 정의된 페이지 ID를 참조
- 사용자가 "Notion 업데이트해줘"라고 요청하면 해당 페이지에 API로 업데이트
- Notion API 사용이 불가한 환경에서는 업데이트할 내용만 텍스트로 제공

## 주의사항

- Notion 페이지 구조가 변경되면 `/CLAUDE.md`의 페이지 ID를 반드시 갱신
- 대량 업데이트 시 API rate limit 고려하여 배치 처리
- Notion API 응답 에러 시 재시도 로직 포함 (429 Too Many Requests 등)
- public 전환 전 민감 정보(토큰, 내부 URL 등) 노출 여부 점검
