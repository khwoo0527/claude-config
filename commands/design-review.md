프로젝트의 UI 코드를 DESIGN.md 기준으로 상세 검증하고 개선을 제안하는 커맨드.

## 인수

`$ARGUMENTS`: 선택. 검토 대상 지정 (예: `src/components`, `특정파일.tsx`, `전체`)
- 미지정 시 변경된 UI 파일을 자동 탐지하여 검토한다.

## 전제 조건

- 프로젝트 루트에 `DESIGN.md`가 있어야 한다.
- 없으면 안내: "DESIGN.md가 없습니다. project-init에서 디자인 시스템을 설정하거나, .claude/templates/designs/에서 골라 프로젝트 루트에 DESIGN.md로 복사하세요."

## 실행 절차

### 1단계: 컨텍스트 로드

1. `/CLAUDE.md`를 읽어 기술 스택과 프로젝트 구조를 확인한다.
2. `/DESIGN.md`를 **전체** 읽어 디자인 시스템을 완전히 숙지한다:
   - 색상 팔레트 (Primary, Secondary, Surface, Border, Semantic)
   - 타이포그래피 계층 (폰트, 크기, 굵기, 행간, 자간)
   - 컴포넌트 스타일 (버튼, 카드, 인풋, 네비게이션)
   - 레이아웃 원칙 (간격 시스템, 그리드, 여백)
   - 깊이/그림자 시스템
   - Do's and Don'ts
   - 반응형 규칙

### 2단계: 검토 대상 결정

- 인수가 있으면 해당 경로의 UI 파일을 검토한다.
- 인수가 없으면:
  - `git diff --name-only`로 변경된 파일 중 UI 파일을 추출한다.
  - UI 파일 판별: `.tsx`, `.jsx`, `.html`, `.css`, `.scss`, `.xaml`, `.Designer.cs` 등
  - 변경 파일이 없으면 프로젝트의 주요 UI 디렉토리를 검토한다.

### 3단계: 디자인 준수 검증

각 UI 파일을 DESIGN.md 기준으로 검증한다:

#### 색상 검증
- 하드코딩된 색상값이 DESIGN.md 팔레트에 있는지 확인
- DESIGN.md에 없는 색상 사용 시 지적
- 의미적 역할에 맞는 색상을 쓰고 있는지 (예: Primary 색상을 보더에 사용하면 안 됨)

#### 타이포그래피 검증
- 폰트 패밀리가 DESIGN.md와 일치하는지
- 크기/굵기 계층이 DESIGN.md 테이블과 일치하는지
- 행간(line-height)이 DESIGN.md 권장값인지

#### 컴포넌트 검증
- 버튼 스타일 (배경, 텍스트색, 패딩, radius, 그림자)
- 카드/컨테이너 스타일 (보더, radius, 그림자)
- 인풋/폼 스타일
- 네비게이션 스타일

#### 레이아웃 검증
- 간격이 DESIGN.md의 간격 시스템(보통 8px 기반)을 따르는지
- 매직넘버 간격이 있는지 (예: 13px, 37px 같은 비체계적 값)
- border-radius가 DESIGN.md 스케일에 있는지

#### Do's and Don'ts 검증
- DESIGN.md의 Don'ts를 위반하는 코드가 있는지

### 4단계: 결과 보고

```
📋 DESIGN.md 준수 리포트

검토 대상: {파일 목록}
기준 문서: DESIGN.md ({brand} 기반)

## 준수 현황

| 영역 | 상태 | 이슈 수 |
|------|------|---------|
| 색상 | ✅/⚠️/❌ | N개 |
| 타이포그래피 | ✅/⚠️/❌ | N개 |
| 컴포넌트 | ✅/⚠️/❌ | N개 |
| 레이아웃 | ✅/⚠️/❌ | N개 |
| Do's/Don'ts | ✅/⚠️/❌ | N개 |

## 이슈 상세

### ❌ Critical (디자인 시스템 위반)
- {파일:줄} — {설명} → {수정 제안}

### ⚠️ Warning (개선 권장)
- {파일:줄} — {설명} → {수정 제안}

### 💡 Suggestion (더 나은 대안)
- {파일:줄} — {설명} → {수정 제안}

## 요약
- 준수율: {N}%
- 수정 필요: {N}개
- 자동 수정 가능: {N}개
```

### 5단계: 수정 제안

사용자에게 묻는다:
- "자동으로 수정할까요?" → Critical/Warning 항목을 자동 수정
- "특정 항목만 수정할까요?" → 선택적 수정
- "보고만 할게요" → 수정 없이 종료

**자동 수정 시 한 파일씩 진행하고 사용자 확인을 받는다.**

## 플랫폼별 검증 범위

> DESIGN.md는 웹 기반으로 작성되어 있다. 플랫폼에 따라 검증 범위를 조정한다.

| 플랫폼 | 검증 범위 |
|--------|----------|
| 웹 | 모든 항목 검증 |
| React Native / Flutter | 색상, 타이포, 간격 검증. 그림자/radius는 플랫폼 변환 고려 |
| WinForms / WPF | 색상 팔레트, 타이포 계층 검증. 컴포넌트/레이아웃은 참고 수준 |
| CLI | 색상 매핑만 검증 |
