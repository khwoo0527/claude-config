세션 시작 시 .claude 의 메모리와 세션 절차를 일괄 로딩하는 부트스트랩 커맨드.

> **왜 필요한가**: Claude Code 는 작업 디렉토리의 `/CLAUDE.md` 와 `~/.claude/projects/{인코딩}/memory/MEMORY.md`(사용자 홈) 만 자동 로딩한다.
> **`.claude/memory/`** (이 폴더의 사용자 룰) 와 **`.claude/rules/workflow/session-init.md`** (세션 절차) 는 자동 로딩 대상이 아니다.
> 이 커맨드가 그 갭을 메우고, 메인 Claude 에게 사용자 작업 스타일·핵심 원칙·세션 절차를 명시적으로 주입한다.

## 인수

`$ARGUMENTS`: 사용하지 않음 (인수 없이 호출 — `/InitLoad`).

## 실행 절차

### 1단계: 메모리 로딩 (MEMORY.md 위임)

1. **`.claude/memory/MEMORY.md`** 를 먼저 Read 한다.
2. MEMORY.md 의 **"로딩 방식"** 섹션에 명시된 절차를 그대로 수행한다.
   - 정책 자체는 InitLoad 가 알지 못한다 — MEMORY.md 가 지시한 대로만 따른다.
   - 정책이 바뀌면 MEMORY.md 만 수정하면 됨 — InitLoad 는 무관.
3. 누락 방지: MEMORY.md 가 지시한 대상이 모두 실제로 Read 되었는지 확인.

### 2단계: 세션 초기화 절차 로딩

**`.claude/rules/workflow/session-init.md`** 를 Read 한다.
- 메모리 외 컨텍스트 (rules, ROADMAP, 작업 유형별 가이드) 의 로딩 절차를 파악한다.
- 작업 유형별 추가 로딩 매트릭스 (코드 구현 / 코드 리뷰 / 문서 작성 등) 를 숙지한다.
- 로딩 누락 시 발생 문제와 Good/Bad 시나리오를 인지한다.

### 3단계: 압축 요약 1줄 보고

사용자에게 로딩 완료를 압축된 한 줄로 보고한다 (토큰 절약 + 룰 능동 적용 신호).

형식:
```
✅ InitLoad: {user 핵심} / {feedback 각 핵심 슬래시 구분} / 세션 절차 OK
```

예시:
```
✅ InitLoad: 1인 개발자(회사+집) / 한 스텝씩 / 문서 위치 가이드 / 품질 타협 X / 테스트 안내 필수 / 위임 신호 시 컨펌 생략 / 세션 절차 OK
```

**원칙**:
- 한 줄로 끝낸다. 섹션 헤더/불릿 금지.
- 메모리 파일 1개당 3~7자 정도의 핵심 키워드만.
- 사용자가 "진짜 읽었나" 의심하지 않을 정도면 충분 — 룰을 다시 가르치는 게 아님.

## 동작 원칙

- **이 커맨드는 매 세션 시작 시 1회 호출**한다 (`/InitLoad`).
- **로딩 후 작업 진행 중**: 메모리 룰을 능동적으로 적용한다 (사용자가 같은 피드백 반복 X).
- **MEMORY.md 의 "로딩 방식" 섹션이 단일 진실 원천** — 정책 변경 시 그 섹션만 수정하면 됨. InitLoad 는 무관.
- **rules/tech/{tech}.md 와 ROADMAP.md 는 로딩하지 않는다** — 작업 시작 시점에 agent (또는 메인 Claude) 가 작업 유형 보고 결정한다.

## 주의사항

- **`/InitLoad` 호출 후 압축 요약을 반드시 보여준다** — 사용자가 무엇이 로딩되었는지 즉시 확인 가능하도록.
- **메모리 파일 Read 누락 금지** — MEMORY.md 가 지시한 로딩 절차의 결과물이 모두 실제로 Read 되었는지 확인.
- **이 커맨드는 메모리 외 컨텍스트 (rules/tech, ROADMAP) 는 로딩하지 않는다** — session-init.md 에 명시된 작업 유형별 절차를 따라 필요 시점에 Read.
- **사용자 홈 메모리 (`~/.claude/projects/{인코딩}/memory/`) 는 시스템이 이미 자동 로딩** — 이 커맨드와 무관.
- **로딩 실패 시**: 누락된 파일을 보고하고 사용자에게 알린다.
