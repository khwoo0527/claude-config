---
name: 추측 금지 — 확인 후 진행
description: "있을 가능성 높음" 같은 추측으로 의사결정/삭제/이관 진행 금지. 항상 정확히 확인 후 진행.
type: feedback
---

규칙: 정보가 다른 곳에 "있을 것 같다"는 추측만으로 삭제·이관·동기화·결정을 진행하지 않는다. 반드시 grep/Read 등으로 **실제 존재 여부를 검증한 후** 진행한다.

**Why:**
- 추측 기반 진행 시 정보 누락 (있는 줄 알았는데 없음 → 손실), 이중 기록 (없는 줄 알았는데 있음 → 중복) 가능성.
- 사용자가 같은 지적을 반복하지 않도록 자체 검증이 기본 태도여야 함.

**How to apply:**
- "있을 가능성 높음", "보통 거기 있음", "있을 거예요" 같은 표현으로 결론 내리지 않는다.
- 의사결정 전 검증 명령 (grep/Read/diff/ls) 으로 실제 상태 확인 후 보고.
- 검증 결과를 사용자에게도 명시: "{파일} L{N} 에 있음" / "{파일} 어디에도 없음" 식으로 근거 제시.
- 손실 우려 작업 (삭제·이관·덮어쓰기) 일수록 검증 단계 생략 금지.
- 빠른 답이 필요해 보여도 **잘못된 빠른 답보다 검증된 답이 더 빠르다** (재작업 비용 회피).
