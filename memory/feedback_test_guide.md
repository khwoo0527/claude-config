## 실행/테스트 시 필수 행동

1. **실행 전에 테스트 가이드를 먼저 제공**
   - 현재 어디까지 동작해야 정상인지 범위 명시
   - 어떤 순서로 테스트하면 되는지 시나리오 제시
   - 예상되는 화면/동작 설명

2. **문제 발견 시 분류**
   - 바로 고칠 것 → 즉시 수정
   - 나중에 할 것 → 일감(이슈)으로 기록, 어느 스프린트에 할지 판단

3. **실행 후 함께 확인하고 넘어가기**
   - 스프린트 끝나고 바로 다음으로 넘어가지 말 것
   - 반드시 실행 → 확인 → 수정/기록 → 다음 진행

**Why:** 사용자는 눈으로 확인하면서 진행하는 스타일. 테스트 가이드 없이 "실행해보세요"만 하면 뭘 확인해야 할지 모름. 실제로 지적받음.
