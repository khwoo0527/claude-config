## 프로필
- 1인 개발자 (바이브 코딩)
- 회사에서도 작업, 집에서도 작업 (2대 환경)

## 환경별 인프라 제약
- 회사 환경: 도커 사용 불가 — 클라우드 인프라(예: Supabase Cloud)로 작업 필요
- 집 환경: 도커 가용 — 로컬 인프라 사용 가능
- 인프라/도구 선택 시 양쪽 환경에서 모두 작업 가능한지 고려한다

## 작업 스타일
- 실행해서 직접 확인하면서 진행하는 스타일
- 결과물을 눈으로 보고 판단
- 급하게 넘어가지 않고 확인 후 다음으로
- .claude 폴더 위주로 지식 저장 (C:\Users 경로 사용 자제)
