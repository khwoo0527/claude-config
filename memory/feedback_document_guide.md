## 정보 위치 가이드
사용자가 정보를 말할 때, 해당 정보가 들어갈 올바른 문서를 적극적으로 안내한다.

| 정보 종류 | 올바른 위치 |
|----------|-----------|
| 기술 스택, 빌드 명령, 프로젝트 구조 | CLAUDE.md |
| 상세 요구사항, 비즈니스 로직, 사용자 시나리오, 장기 목표 | docs/prd.md |
| Phase/Sprint 구조, 일정 | ROADMAP.md |
| Sprint 실행 명세 | docs/sprint/sprint{N}.md |

**Why:** CLAUDE.md에 PRD 수준 상세를 넣으면 비대해져서 바이브 코딩 품질이 떨어짐. 실제로 위반하여 수정함.

## 메모리 저장 위치
중요한 피드백/학습이 생기면, 저장 전에 어디에 저장할지 판단하고 사용자에게 확인한다.

| 종류 | 위치 | 이유 |
|------|------|------|
| 범용 (모든 프로젝트 적용) | `.claude/memory/` | git에 포함, 다른 프로젝트로 복사됨 |
| 프로젝트 종속 | 프로젝트 메모리 | 이 프로젝트에서만 유효 |

판단이 애매하면: "이 내용은 범용 가이드인데, .claude/memory에 저장해서 다른 프로젝트에서도 쓸 수 있게 할까요?" 사용자에게 확인.

## .claude/ 폴더 종속성 금지
- `.claude/rules/`, `.claude/memory/` 등 범용 파일에 **프로젝트 종속 내용(프로젝트명, 특정 기능명, 특정 화면명 등)을 절대 넣지 않는다**
- 예시를 넣을 때도 범용적인 예시만 사용한다
- .claude 폴더는 다른 프로젝트에 그대로 복사해서 쓸 수 있어야 한다

**Why:** .claude 폴더에 특정 프로젝트 기능명을 예시로 넣으면 다른 프로젝트에서 맥락이 안 맞음. 실제로 위반하여 지적받음.
