---
name: 위임 신호 받으면 그 작업 동안 컨펌 생략하고 알아서 진행
description: 사용자가 "권한 다 OK" / "묻지 말고 진행" / "알아서 써" / "자리 비울게" 등 명시적 위임 신호 주면 그 작업이 끝날 때까지 모든 컨펌/권한 확인 생략
type: feedback
---

규칙: 사용자가 작업 진입 시점에 위임 신호를 주면 그 작업 동안은 다음 모두 컨펌 없이 진행한다:
- DB 작업 (TRUNCATE, ALTER, CREATE/DROP, INSERT/DELETE 등)
- 파일 시스템 작업 (대량 이동/삭제, 권한 변경)
- 빌드/배포 (msbuild, pg_dump, dotnet build 등)
- 평소 "이거 해도 돼요?" 묻던 작업 전부
- "옵션 A/B/C 중 어느 거?" 식의 작은 선택도 합리적 기본값으로 알아서 결정

위임 신호 예시:
- "권한 다 사용해도 좋아" / "권한 다 OK"
- "묻지 말고 알아서 진행해"
- "자리 비울게 / 자리 비울 거라"
- "그냥 가봐 / 그냥 진행해줘"
- "너가 결정해 / 너 알아서"

**Why:** 사용자가 자리 비우거나 빠른 진행 원할 때 매 단계 컨펌 받으면 작업이 멈춤 → 사용자 시간 낭비 + 흐름 끊김. 명시적 위임이 곧 사전 동의로 작용.

**How to apply:**
- 위임 신호 받은 작업 동안만 적용. 새 작업 시작 시 다시 일반 룰로 복귀.
- 작업 끝나면 결과 보고 필수 (사용자가 자리 돌아왔을 때 즉시 컨텍스트 복구되도록).
- 데이터 삭제/롤백 불가 작업도 위임 후라면 진행. 단 사후 보고 명확히.
- 의도 불분명하거나 위임 신호 없으면 일반 룰 우선 (시스템 작업/인증 정보 등은 사전 확인 룰 따름).
- 신호의 범위도 추론: "이번 작업" 이 어디까지인지 자연스럽게 해석 (예: 특정 기능 이식 작업 동안이면 그 전체).
