## 품질 절대 타협 금지
범용화는 전문성 희석이 아니다. 어떤 기술 스택이든 해당 분야 시니어 개발자 수준의 전문가로 동작해야 한다.

- rules/{tech}.md는 깊은 전문 지식을 담아야 함 (얕은 가이드라인 X)
- 코드 리뷰, 구현, 설계 모두 최고 수준 품질 기준 적용
- "범용적으로 작성" ≠ "두루뭉술하게 작성"

**Why:** 범용화 목적은 "어떤 프로젝트에서든 최고 품질"이지 "평균적으로 무난한 품질"이 아님.

## 에이전트 절차 준수
에이전트가 설계된 작업(ROADMAP 생성, Sprint 계획 등)을 대화 중에 직접 수행할 때, 해당 에이전트의 구조화된 절차를 반드시 따른다.

1. 해당 에이전트 문서를 먼저 읽는다
2. 단계별 절차를 순서대로 따른다
3. 품질 검증 체크리스트를 마지막에 반드시 수행한다
4. 템플릿/출력 형식을 그대로 사용한다
5. 대화에서 얻은 맥락을 추가로 반영한다

**Why:** 자유롭게 작성하면 단계를 건너뛰거나 품질이 들쭉날쭉해짐.

## .claude 폴더는 git에 포함
.claude/ 폴더는 반드시 git에 포함되어야 한다. .gitignore에 넣지 않는다.

- 토큰 등 민감 정보가 포함된 경우, gitignore가 아닌 다른 안전한 방법을 제안
- 환경 동기화(회사/집)가 최우선 가치

**Why:** 사용자가 멀티 환경에서 작업하며, .claude가 없으면 다른 환경에서 고생함. 실제로 겪음.
